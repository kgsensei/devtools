Setup/Installation

Since the inputs library requires the base custom.css library, you'll need to add both. In the head tag, just add these lines of code: 
<style>
    :root {
        --kg-bg: rgb(25, 32, 40);
        --kg-fg: rgb(225, 225, 245);
        --kg-fg-darker: rgb(190, 190, 190);

        --kg-primary: rgb(30, 172, 244);

        --kg-font: Helvetica, sans-serif;
    }
</style>
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/kgsensei/devtools@1.1.1/custom.min.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/kgsensei/devtools@1.1.1/custom.input.min.css">
We highly recommend learning more about the custom.css library before using the inputs extension.

Adding An Input

Unlike most of the custom.css libraries, the input library does most of the styling for you, by default! 
<input id="exampleInput" type="text" placeholder="username">
Buttons aren't effected by the input library so you'll need to style them with the base custom.css library classes. 
<form>
    <input id="exampleInput" type="text" placeholder="username">
    <button class="btn btn-outline primary" type="submit">Submit</button>
</form>

Label Styles

Labels are a nice addition to any form, in the inputs library we don't have many options for them yet...

If you want your label to sit closer to the top of the input itself use the 
.label-low
class.
